# Merge Listed and Clinical Pools

将已完成各自清洗与疾病 Mapping 的“中国已上市库”和“中国临床库”，按统一表头合并，并通过可审计的跨库关系判断，生成最终分子池。

最终研究单元通常为：**药品资产或固定复方 × 剂型 × 治疗疾病**。

## What it does

- 对两个来源库进行只读检查，识别表头、主键、必需字段、Disease 多结果、库内重复、公式和外部链接等风险；
- 以当前所有 TA 共用的 21 列表头为起点，支持按经确认的 TA 特例增删、改名和调整字段顺序；
- 将上市库“持证商”和临床库“申办方”统一映射到 `License Holder`，同时在原始表保留各自定义；
- 直接承接上游已形成的单列“是否VBP”，本 Skill 不重新计算 VBP；
- 将已上市数据置于上半部分、临床数据置于下半部分，生成统一字段的合并底稿；
- 综合资产名称、通用名、成分、剂型、治疗疾病和相关适应症，生成高召回的跨库候选关系；
- 支持“一条临床记录对应多个已上市候选”的判断与审计；
- 将人工确认的保留、上市库已覆盖、非治疗性资产、非固定联合方案和范围外历史上市等决策结构化回填；
- 在所有判断关闭后执行保留、排除、Disease 修正或资产名称修正，并生成连续编号的最终分子池；
- 对行数、来源构成、序号、承接关系、原始表完整性和待确认项进行最终 QC。

## Workflow

1. **输入与预检**：提供 TA、两个来源文件及 Sheet、研究范围、Disease 口径和输出位置，先进行只读检查。
2. **表头确认**：基于推荐字段形成当次 Run Schema；字段数量不是固定值，必须经人工确认后冻结。
3. **底稿合并**：按确认后的字段映射合并两个来源库，保留原始数据表，并在内部审计信息中保存来源标识，不额外占用 21 列主表字段。
4. **候选关系生成**：为每条临床记录生成可能对应的已上市候选；候选结果只用于复核，不直接触发删除。
5. **集中判断**：集中确认 Disease、适应症、剂型、化学形式、固定复方、非治疗性用途及上市库缺口等例外。
6. **预执行检查**：确认每条临床记录只有一个一致、已关闭的最终动作，并展示预计最终行数。
7. **正式出表**：获得明确执行确认后生成最终工作簿，并重新打开完成 QC。

## Output

最终工作簿包含：

- **最终分子池**：采用当次确认的 Run Schema，保留静态值和连续唯一分子序号；
- **跨库关系判断**：保留候选关系、比较依据、人工结论、证据、动作代码、承接上市行及最终去向；
- **中国已上市原始表**：保持原始单元格内容和顺序，供追溯与核验；
- **中国临床原始表**：保持原始单元格内容和顺序，供追溯与核验。

## Key safeguards

- 不修改、覆盖或删除任何输入文件、中间文件或已批准交付物；
- 未确认当次最终表头前，不生成正式合并底稿；
- 不因名称相似或“中国阶段=已上市”而自动删除临床记录；
- 候选关系采用高召回策略，但任何候选都不等同于最终重复结论；
- 不跨不同 Disease 直接合并，必须结合相关适应症判断是真实新增还是 Mapping 差异；
- 新 Disease、新剂型、新研究分子、固定复方或具有实质差异的化学/释放形式原则上保留，除非已有明确规则或人工结论；
- 删除“已被上市库覆盖”的临床行时，必须指向一个或多个有效的已上市承接行；
- 同一临床记录存在多个候选时，所有候选行必须使用一致的最终动作和修正值；
- 存在空白动作、冲突动作、`HOLD`、待确认状态或无效承接行时，禁止生成最终分子池；
- 最终行数必须满足“已上市输入 + 临床输入 − 已覆盖临床行 − 已排除临床行”的核算关系；
- 心血管数据只用于回归测试，其数量、比例和具体决策不会被用作其他 TA 的默认标准。

## Boundaries and limitations

- 输入的两个库应已完成各自的上游清洗，并使用同一版本的 TA Disease 清单；
- 本 Skill 不负责重新拆分适应症、全面重做医学判断或重新建立 Disease Mapping；
- 本 Skill 不自动解决来源库内部的重复记录；发现未经确认的库内重复时会暂停；
- 本 Skill 不执行 BI 自有产品标记、MOA 去重、VBP 核减、Giant MNC 标记或商业优先级筛选；
- 名称和成分匹配只能生成复核候选，不能替代医学、药学或监管数据库结论；
- 盐型、立体异构体、游离酸/酯、缓控释差异、固定复方属性和新增适应症等情况，可能需要人工判断或专业数据库补充；
- “已上市但上市库无直接候选”的记录不能自动删除，需要核验治疗属性、首次上市时间、剂型和获批适应症；
- 当前 21 列是跨 TA 公共基线；如某个 TA 需要例外字段，仍须重新确认最终字段及映射；
- 所有开放性判断必须关闭后才能正式出表，因此结果质量仍依赖输入数据质量和人工确认质量。

## Typical use case

适用于某一 TA 已分别获得“中国已上市”和“中国临床”分子池，且两边已完成基础清洗和 Disease Mapping，需要统一表头、识别跨库重合与新增价值，并生成可追溯、可复核、可继续用于后续筛选分析的最终分子池的场景。
